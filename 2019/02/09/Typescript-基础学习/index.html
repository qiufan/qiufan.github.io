Typescript 基础学习

发表于 2019-02-09 | 分类于 基础学习

为了写个小程序，正好以后入职也是要接触Typescript，先稍稍学习下基础，主要学习于官方文档。

基础类型

变量 代码表示 例子 备注
布尔 boolean false,true 无
数字 number 0x,0b,0o 只有浮点，支持十，十六，二，八进制
字符串 string (用””或者’’包裹，也可以用’’包裹，${expr}嵌入)
数组 type[],Array
元组 [type1,type2] [string,name] 用数组的形式包括两个类型就是元组，可以越界访问，会自动用联合类型替代
枚举 enum enum Color {Red=1,Green,Blue} 可以手动赋予初始值
Any any any[]=[1,true,”free”] 可以视作任意，包括可以在上面调用函数
void undefined,null
null和undefined 默认情况它们是所有类型的子类型，–strictNullChecks可以规定只能赋给void和它们自己，同时使用联合类型
Never never function error(message: string): never 永远不存在，必须在函数无返回，或者报错
object 非原始类型 除了number,string，boolean,symbol,null和undefined之外的类型

类型断言的方式有两种,someValue或者someValue as string

变量声明

let 和 const

let

let的使用跟c++很像，不能重复定义，块级有效，可以覆盖，块级与变量环境绑定的，即使代码已经运行过去，环境依旧存在
‘’’
function theCityThatAlwaysSleeps() {
let getCity;

if (true) {
    let city = "Seattle";
    getCity = function() {
        return city;
    }
}

return getCity();

}
‘’’
正如这里的例子，即使city已经执行完毕，return getCity()时依旧可以访问。

const

const的特性与let一致，就是不能变动，维持不同的常量

解构和展开

展开比较容易理解
‘’’
let first = [1, 2];
let second = [3, 4];
let bothPlus = [0, …first, …second, 5];
‘’’
展开对象也可以，后面会覆盖钱前面相同属性，同时会丢失方法，只会展开可枚举的属性
‘’’
let defaults = { food: “spicy”, price: “$$”, ambiance: “noisy” };
let search = { …defaults, food: “rich” };
‘’’
解构的理解，看了点其他博客，理解不够深入，先摘要写一下，以后有机会再来补充。
在开发过程中，我们经常遇到这样问题，需要将对象某个属性的值赋给其它两个变量代码如下：
‘’’
var person = {
name:’xxxx’,
age: 20,
hobby:[‘football’, ‘run’]
}

var a = person.name;
var hobby = person.bobby;
‘’’
有了解构，我们可以精简代码了
‘’’

var person = {
name:’xxxx’,
age: 20,
hobby:[‘football’, ‘run’]
}
let {a, hobby} = person;

let [first] = person.hobby;
‘’’
数组解构
‘’’
let input = [1, 2];
let [first, second] = input;
console.log(first); // outputs 1
console.log(second); // outputs 2
‘’’

接口

因为之前一直没有实际用过接口，我目前对于接口的理解是一种定义和规定，接口本身不具有意义
‘’’
interface LabelledValue {
label: string;
}

function printLabel(labelledObj: LabelledValue) {
console.log(labelledObj.label);
}

let myObj = {size: 10, label: “Size 10 Object”};
printLabel(myObj);
‘’’
满足比较条件就可以了，多的属性被忽略
‘’’
interface SquareConfig {
color?: string;
width?: number;
}
‘’’
可选属性用?号表示
‘’’
interface Point {
readonly x: number;
readonly y: number;
}
‘’’
readonly来表示只读，然而可选属性与多的属性结合会出错。
‘’’
interface SquareConfig {
color?: string;
width?: number;
}

function createSquare(config: SquareConfig): { color: string; area: number } {
// …
}

let mySquare = createSquare({ colour: “red”, width: 100 });
‘’’
上面这段colour是不行的，对象字面量会被特殊对待而且会经过额外属性检查，类型断言可以绕开检查，也可以添加索引签名。
接口也可以表示函数类型
‘’’
interface SearchFunc {
(source: string, subString: string): boolean;
}
‘’’
可索引类型具有一个索引签名，它描述了对象索引的类型，还有相应的索引返回值类型。
‘’’
interface StringArray {
[index: number]: string;
}

let myArray: StringArray;
myArray = [“Bob”, “Fred”];

let myStr: string = myArray[0];
‘’’
TypeScript支持两种索引签名：字符串和数字。 可以同时使用两种类型的索引，但是数字索引的返回值必须是字符串索引返回值类型的子类型。
‘’’
class Animal {
name: string;
}
class Dog extends Animal {
breed: string;
}

// 错误：使用数值型的字符串索引，有时会得到完全不同的Animal!
interface NotOkay {
[x: number]: Animal;
[x: string]: Dog;
}
‘’’
